```python
# 課題1: 業界内での偏差値（業界ごとに平均・SDを取り直す）
cust['業界内偏差値'] = cust.groupby('業界')['スコア'].transform(
    lambda s: 50 + 10*(s - s.mean())/s.std(ddof=0))
print(cust[['顧客ID','業界','偏差値','業界内偏差値']].head())
# → 全体偏差値は「全顧客の中での位置」、業界内偏差値は「同業他社の中での位置」。
#    業界ごとに購買規模が違うので、公平に比べたいときは業界内が有効。

# 課題2: 累計売上を2倍重視した合成スコア
cust['スコア2'] = Z['購入回数'] + 2*Z['累計売上'] - Z['Recency']
print(cust.sort_values('スコア2', ascending=False)[['顧客ID','スコア','スコア2']].head())
# → 大口顧客が上位に来やすくなり、トップ5の顔ぶれが入れ替わる。
```

**課題3（考察）**：累計売上のような **外れ値（極端な大口）** が混じるデータでは、平均・SDが外れ値に引っ張られ、Zスコアが歪む。中央値と IQR（四分位範囲）で `(x − 中央値)/IQR` のように標準化すると外れ値に強い。少数の超大口がいるBtoB売上などで有効。
