# 完了ログのセットアップ（講師用）

各ノート末尾の「完了ログ」セルは、学習者が実行すると **学習者コード・ノート名・日時** を Google スプレッドシートに1行追記する。学習者は名前を毎回入力せず、**コードを最初に一度だけ設定**すれば以降は全ノートで自動送信される。コードと実名の対応は講師が非公開の名簿で管理する。

## 仕組み
学習者のノート → `LOG_URL`（Apps Script Web App）へ POST（`token`＋`code`）→ 検証・サニタイズ後にスプレッドシートへ追記。

> 学習者コードの取得順（混在環境に対応）：**Colabシークレット `STUDENT_CODE` → 環境変数 `STUDENT_CODE` → 保存ファイル `~/.student_code` → 初回に画面入力（保存）**。一度設定すれば再入力は不要。

## 手順

### 1. スプレッドシートと Apps Script を用意
1. 新しい Google スプレッドシートを作る。
2. 拡張機能 → Apps Script を開き、以下を貼り付けて保存。

```javascript
function doPost(e) {
  try {
    if (!e || !e.postData || !e.postData.contents) return json_({ok:false, error:'no body'});
    if (e.postData.contents.length > 2000) return json_({ok:false, error:'too large'});

    var d;
    try { d = JSON.parse(e.postData.contents); }
    catch (_) { return json_({ok:false, error:'bad json'}); }

    // 共有トークンで最低限のゲート（下の 2 で LOG_TOKEN を設定）
    var expected = PropertiesService.getScriptProperties().getProperty('LOG_TOKEN');
    if (expected && d.token !== expected) return json_({ok:false, error:'unauthorized'});

    var code       = clean_(d.code);       // 学習者コード
    var notebook   = clean_(d.notebook);
    var clientTime = clean_(d.time);

    var lock = LockService.getScriptLock();     // 並行書き込みを直列化
    lock.waitLock(5000);
    try {
      var ss = SpreadsheetApp.getActiveSpreadsheet();
      var sheet = ss.getSheetByName('log') || ss.insertSheet('log');
      sheet.appendRow([new Date(), clientTime, code, notebook]);  // 先頭にサーバ受信時刻
    } finally {
      lock.releaseLock();
    }
    return json_({ok:true});
  } catch (err) {
    return json_({ok:false, error:'server error'});  // 詳細は外部に返さない
  }
}

// 文字列化 + 長さ制限 + 制御文字除去 + 数式インジェクション無効化
function clean_(v) {
  if (v === undefined || v === null) return '';
  var s = String(v).slice(0, 200).replace(/[\r\n\t]/g, ' ');
  if (/^[=+\-@]/.test(s)) s = "'" + s;   // 先頭が数式扱いの文字なら ' を前置
  return s;
}

function json_(obj) {
  return ContentService.createTextOutput(JSON.stringify(obj))
    .setMimeType(ContentService.MimeType.JSON);
}
```

### 2. トークンを設定
- プロジェクトの設定 → スクリプト プロパティ で `LOG_TOKEN` に**長いランダム文字列**を設定（例: `openssl rand -hex 16`）。未設定だと誰でも書けるため必ず設定する。

### 3. Web App としてデプロイ
- デプロイ → 新しいデプロイ → 種類「ウェブアプリ」。「実行=自分」「アクセス=全員」。
- 発行された **ウェブアプリ URL**（`.../exec`）をコピー。これが `LOG_URL`。

### 4. 学習者コードを発行（名簿を作る）
学習者ごとに一意コードを発行し、`code↔実名` の**非公開名簿**を作る。

```python
import secrets, csv
names = ["山田太郎", "佐藤花子"]        # ← 学習者名
with open('roster.csv', 'w', encoding='utf-8', newline='') as f:
    w = csv.writer(f); w.writerow(['code', 'name'])
    for nm in names:
        w.writerow([secrets.token_hex(4), nm])   # 例: 3f9a1c2b
print('roster.csv を作成。各学習者に自分の code だけを個別に伝える。roster.csv は非公開で保管する。')
```

- `roster.csv` の中身を、集計用スプレッドシートの **`roster` タブ**（列: `code`, `name`）にも貼っておくと、後で名前に変換できる。
- 各学習者には**自分のコードのみ**を個別（メール/DM等）で伝える。コードを共有しない限り本人性が保たれる。

### 5. 全ノートに URL とトークンを設定して配布
リポジトリのルート（または配布用コピー）で次を実行。

```python
import nbformat, glob
url   = "https://script.google.com/macros/s/XXXX/exec"
token = "2で設定したのと同じトークン"
for f in glob.glob('**/*.ipynb', recursive=True):
    if '/.' in f: continue
    nb = nbformat.read(f, as_version=4)
    for c in nb.cells:
        if c.cell_type == 'code' and 'LOG_URL = ""' in c.source:
            c.source = (c.source
                        .replace('LOG_URL = ""',   f'LOG_URL = "{url}"')
                        .replace('LOG_TOKEN = ""', f'LOG_TOKEN = "{token}"'))
    nbformat.write(nb, f)
```

> **公開リポジトリに注意**：URL・トークンを入れたノートを公開リポジトリにコミットしない。設定は配布物にのみ行い、公開側は空欄（`LOG_URL = ""` / `LOG_TOKEN = ""`）のまま残す。配布は private リポジトリ・zip・Classroom など非公開経路で。

### 6. 学習者への案内
- 配布したコードを **一度だけ** 設定してもらう:
  - **Colab**：左の鍵アイコン（シークレット）で `STUDENT_CODE` に自分のコードを登録 → 全ノートで自動。
  - **ローカル**：環境変数 `STUDENT_CODE` を設定（`export STUDENT_CODE=...`）、または初回にノートの入力欄へ貼ると `~/.student_code` に保存され次回から不要。
- あとは各ノートの最後のセルを実行するだけで記録される。

## 集計
- `log` シートに `受信時刻 / 申告時刻 / コード / ノート名` が溜まる。
- `roster` タブとコードで突き合わせ（VLOOKUP や QUERY）して実名に変換 → ピボットで「名前 × ノート名」の進捗一覧。

## セキュリティ上の注意
- Web App は「全員アクセス可」で公開するため、URL・トークンが漏れれば第三者も書き込める。トークン・サイズ上限・数式インジェクション対策（`clean_`）でリスクを抑えているが、厳密運用なら Workspace ドメイン限定＋「アクセスするユーザーとして実行」（学習者ログイン必須）に切り替える。
- 学習者コードは本人性の要。共有されると別人になりすませるため、コードは個別配布し、必要に応じて定期的に更新する。
- 記録されるのはコード・ノート名・日時のみ（実名はログに残らず、名簿側でのみ突合）。名簿とスプレッドシートの共有範囲を最小化する。
