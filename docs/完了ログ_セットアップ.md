# 完了ログのセットアップ（講師用）

各ノート末尾の「完了ログ」セルは、学習者が実行すると **名前・ノート名・日時** を Google スプレッドシートに1行追記する。学習者は Google 認証不要（下記の Web App URL にトークン付きで POST するだけ）。

## 仕組み
学習者のノート → `LOG_URL`（Apps Script Web App）へ POST（`token` 付き）→ 検証・サニタイズ後にスプレッドシートへ追記。

> セキュリティ上の割り切り：学習者のGoogle認証を不要にするため Web App を「全員アクセス可」で公開する。そのため URL・トークンは配布ノートに埋め込まれ、学習者からは見える“弱い秘密”になる。下記のトークン・入力検証・上限で、無差別スキャンや数式インジェクションといった実害を抑える。

## 手順

### 1. スプレッドシートと Apps Script を用意
1. 新しい Google スプレッドシートを作る。
2. 拡張機能 → Apps Script を開き、以下を貼り付けて保存。

```javascript
function doPost(e) {
  try {
    if (!e || !e.postData || !e.postData.contents) return json_({ok:false, error:'no body'});
    if (e.postData.contents.length > 2000) return json_({ok:false, error:'too large'});

    var d;
    try { d = JSON.parse(e.postData.contents); }
    catch (_) { return json_({ok:false, error:'bad json'}); }

    // 共有トークンで最低限のゲート（下の 2 で LOG_TOKEN を設定）
    var expected = PropertiesService.getScriptProperties().getProperty('LOG_TOKEN');
    if (expected && d.token !== expected) return json_({ok:false, error:'unauthorized'});

    var name       = clean_(d.name);
    var notebook   = clean_(d.notebook);
    var clientTime = clean_(d.time);

    var lock = LockService.getScriptLock();     // 並行書き込みを直列化
    lock.waitLock(5000);
    try {
      var ss = SpreadsheetApp.getActiveSpreadsheet();
      var sheet = ss.getSheetByName('log') || ss.insertSheet('log');
      sheet.appendRow([new Date(), clientTime, name, notebook]);  // 先頭にサーバ受信時刻
    } finally {
      lock.releaseLock();
    }
    return json_({ok:true});
  } catch (err) {
    return json_({ok:false, error:'server error'});  // 詳細は外部に返さない
  }
}

// 文字列化 + 長さ制限 + 制御文字除去 + 数式インジェクション無効化
function clean_(v) {
  if (v === undefined || v === null) return '';
  var s = String(v).slice(0, 200).replace(/[\r\n\t]/g, ' ');
  if (/^[=+\-@]/.test(s)) s = "'" + s;   // 先頭が数式扱いの文字なら ' を前置
  return s;
}

function json_(obj) {
  return ContentService.createTextOutput(JSON.stringify(obj))
    .setMimeType(ContentService.MimeType.JSON);
}
```

### 2. トークンを設定
- Apps Script の「プロジェクトの設定」→ スクリプト プロパティ で、`LOG_TOKEN` に**長いランダム文字列**を設定（例: `openssl rand -hex 16` の出力）。
- 未設定なら誰でも書ける状態になるため、必ず設定する。

### 3. Web App としてデプロイ
- デプロイ → 新しいデプロイ → 種類「ウェブアプリ」。
- 「次のユーザーとして実行」= **自分**、「アクセスできるユーザー」= **全員**。
- 発行された **ウェブアプリ URL**（`https://script.google.com/macros/s/XXXX/exec`）をコピー。これが `LOG_URL`。

### 4. 全ノートに URL とトークンを設定
リポジトリのルートで次を実行すると、全ノートの `LOG_URL` と `LOG_TOKEN` を一括設定できる。

```python
import nbformat, glob
url   = "https://script.google.com/macros/s/XXXX/exec"   # ← 3でコピーしたURL
token = "2で設定したのと同じトークン"
for f in glob.glob('**/*.ipynb', recursive=True):
    if '/.' in f: continue
    nb = nbformat.read(f, as_version=4)
    for c in nb.cells:
        if c.cell_type == 'code' and 'LOG_URL = ""' in c.source:
            c.source = (c.source
                        .replace('LOG_URL = ""',   f'LOG_URL = "{url}"')
                        .replace('LOG_TOKEN = ""', f'LOG_TOKEN = "{token}"'))
    nbformat.write(nb, f)
```

設定後に配布すれば、学習者は `NAME` を入れて最後のセルを実行するだけで記録される。

> **公開リポジトリに注意**：URL・トークンを入れたノートを**公開リポジトリにコミットしない**こと（エンドポイントが露出する）。設定は配布物にのみ行い、公開リポジトリには空欄（`LOG_URL = ""` / `LOG_TOKEN = ""`）のまま残す。配布は private リポジトリ・zip・Classroom など非公開経路で行う。

## 集計
- `log` シートに `受信時刻 / 申告時刻 / 名前 / ノート名` が溜まる。
- ピボットテーブルで「名前 × ノート名」を作れば、誰がどこまで進めたかを一覧できる。

## セキュリティ上の注意
- Web App は「全員アクセス可」で公開するため、URL とトークンが漏れれば第三者も書き込める。上記トークン・サイズ上限・数式インジェクション対策（`clean_`）でリスクを抑えているが、厳密に守るなら Workspace ドメイン限定＋「アクセスするユーザーとして実行」（学習者ログイン必須）に切り替える。
- サーバ側の `clean_` で先頭 `= + - @` を無効化しているため、名前欄に数式を入れても実行されない。
- 記録されるのは名前・ノート名・日時のみ（PII は氏名）。シートの共有範囲を最小化し、保持・削除方針を配布時に周知する。
